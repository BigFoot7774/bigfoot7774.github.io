# 실패한 원인

디자인이 너무 허접하고 내가 원하는 만큼의 홈페이지를 만들기 위해서는
javascript와 css기초에 대해 더 깊이 있게 배울 필요가 있었다고 느낌

# 이후 계획

1. 새로 홈페이지를 디자인하면서 javascript와 css에 대해 더 깊이 있게 공부

2. 기초지식에 대한 이해가 부족하다보니 라이브러리들을 가져와서 사용할 때 발생하는 에러에 대처가 어려웠음.
그래서 index홈페이지 디자인은 no library, no framework를 지향하기로 함

3. 이미지도 마찬가지로 웬만하면 내가 조작하기 편할만큼 단순한 디자인을 사용하기로 함
(예시로 발바닥 모양 index이미지는 엑셀로 만듦)

3. [프로젝트 목표 설정] 단순한 게시판이 아니라 디스커스와 같이 게시판과 댓글 서비스를 사용자들에게 제공하여
깃허브같은 정적사이트에서도 블로그 포스팅이 가능하게 해주는 서비스를 만들기로 함

4. [프로젝트 목표 설정] 자바스크립트로
History API(pushstate)와 ajax를 접목시킨 라이브러리를 만들어서
깃허브 블로그같은 정적사이트에서도 RESTful한 URI를 제공해주자

5. index 페이지는 github에서 무료로 제공해주는 호스팅을 사용.
index 페이지 외에 REST api를 사용하는 스프링 프로젝트를 만들고
서버는 집에 남는 넷북에 리눅스로 서버 설치예정

2019-8-28 [프로젝트 목표 수정]

# 수정한 이유

pjax 라이브러리를 만들어 깃허브페이지에서 REST API를 만들게 되면
페이지 새로고침 시나 뒤로가기, 혹은 URI로 사용자가 직접 접근 시에
정적페이지에서는 페이지를 정상적으로 로드하는 것이 불가능하다.
물론 hashbang(#!)이나 History.pushstate와 같이 uri를 조작하는 것은 가능하다.
하지만 해쉬뱅은 자바스크립트로 구동 되기때문에
자바스크립트 에러 발생 시 모든 동작들이 중지되어 웹의 기본에 위배되기 때문에 보류했다.
pushstate는 좀 더 공부를 해봐야 확신이 생기겠지만
지금 본인의 실력으로 내린 결론으로는 서버사이드렌더링이 아닌 이상
REST를 구현하기는 어려울 것으로 판단이 되어 이 부분도 보류하게 되었다.

# 이후 계획

깃허브 페이지는 싱글 URL + 페이지에 ajax로만
돌아가는 반응형 웹애플리케이션(간단한 자기소개 홈페이지)로 마무리하고
새로운 스프링 웹프로젝트를 시작하여 이 프로젝트 디자인에 이어서 작업을 진행할 계획
